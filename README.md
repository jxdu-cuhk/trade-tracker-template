# 韭菜账本 Leek Ledger

一个本地优先的交易记录、盈亏复盘与持仓看板。

记录每一次被市场教育的痕迹。

特别适合同时管理 A 股、港股、美股、期权等多币种、多账户交易的人，用一个本地工作簿统一记录、折算和复盘。

韭菜账本可以手动录入交易，也可以把券商数据交给 Codex、Claude 等 AI 工具清洗后导入：首次建账适合使用券商导出的 `.csv` / `.xlsx` 历史成交记录，后续补单则可以直接把完整订单截图发给 AI 工具追加录入。

这个公开版本只保留可运行框架和合成示例工作簿，不包含任何真实交易记录、券商导出、标的缓存或个人数据。示例数据只用于展示看板外观，克隆后可以直接清空 `交易记录` 工作表再录入自己的交易。

## 现在能看什么

看板以 `Trade Tracker.xlsx` 为唯一数据源，刷新后生成 `Trade Tracker.html`。

主要页面包括：

- 当前持仓：按持仓绝对值排序，展示持仓总资产、总盈亏、总市值、当日持仓浮盈变动、已实现盈亏等；同页会显示资金口径 / 数据质量，说明账户资产、风险敞口、期权资金兜底和行情完整度。
- 未平仓期权：展示 short put / covered call 等未平仓期权，匹配现价、浮动盈亏、占用本金和到期信息。
- 盈亏日历 / 阶段账单：按日期和区间看已实现盈亏，也可以把日历切到持仓浮盈或合计口径；已实现明细支持按交易标签筛选，最新交易日浮盈与当前持仓页汇总保持一致，历史日期沿用总收益曲线日线点。
- 清仓分析：复盘已清仓标的和交易结果。
- 期权收益分析：拆分期权相关收益、归因和现金占用。
- 总体概览：汇总资产、盈亏、交易费用和分币种口径。
- 总收益曲线：看账户曲线、市场基准、超额收益和 K 线视图。
- 收益报告、分年度个股汇总、交易时间线、工作表入口等辅助页面；交易时间线支持标签列和标签筛选，分年度个股汇总按年度切片，跨年持仓只计算所选年度内的盈亏。

网页顶部按业务分成 `持仓 / 收益 / 复盘 / 明细 / 全部`，每页会组合相关区块；当前分页里的第一个区块会作为主工作区展示，后面的区块作为辅助信息收轻，避免每张卡片都抢主次。分页下方会随当前页面显示区块快捷按钮，可以直接跳到清仓、期权、时间线等长页面里的下游区块。“刷新”和“统一口径币种”在同一条操作区，刷新进度默认是轻量状态条，只有刷新中或报错时展开步骤。

显示数据接口和当前口径来源整理在 `.trade-tracker/DATA_INTERFACES.md`。当前已新增统一展示层 `display_payload.py`，持仓总计、分币种折算、最新日浮盈、已实现交易列表、已实现日汇总、交易标签、资金口径和数据质量会先在那里算好，再给页面区块复用。

## 盈亏口径

看板里的 `总盈亏` 按交易事实复算：已平仓现股/期权盈亏 + 分红净额 + 当前未平仓仓位的纯浮盈浮亏。当前持仓表只会把“本轮持仓起始日之后”的部分卖出收益、已平仓期权收入和分红净额回冲到 `持仓成本`，用于处理 T 操作；如果某个标的曾经清仓归零，清仓前的历史收益不会滚进新一轮持仓的 `浮动盈亏`。

分年度个股汇总里，`已实现盈亏` 只显示交易平仓盈亏，`分红净额` 单独展示股息和扣税，`总盈亏` 再把两者和期末浮盈合起来。券商原始成交明细只会在对应导入汇总行的开仓到平仓日期窗口内回补，避免同一代码后续周期被拿来重复替换。

核验时优先看这条不变量：逐行平仓公式无误后，按标的汇总的 `总盈亏` 应该等于 `已实现 + 分红 + 当前未平仓纯浮盈`，总收益曲线最后一个点也应和总体概览的分币种 `总盈亏` 对齐。

## 总收益曲线

总收益曲线支持这些交互：

- 支持 `汇总 / A股 / 港股 / 美股` 切换，每次只展示一个范围。
- 支持统一口径切换：人民币、港币、美元。
- 基准指数覆盖 A 股、港股、美股常见指数，包括上证、深证、创业板、上证50、沪深300、中证500、科创综指、科创50、恒指、恒科、国企、标普500、纳指、道指、罗素2000。
- 指数历史行情按指数长期缓存；刷新时只加载一次缓存、只补缺失日期和当天实时尾点，最后一次性落盘，避免每个指数重复读写缓存。
- 科创综指会从官方发布起点 `2022-04-11` 开始，早期腾讯缺口优先走中证指数官方接口短超时补全，不把东方财富作为科创综指主来源。
- 支持收益率和盈亏金额两种口径。
- 支持超额收益曲线，用账户收益减去当前 baseline。
- 账户收益率按日频 TWR 思路计算：每天单独计算收益率，再逐日连乘；净买入/卖出会作为外部资金流处理，避免把入金误算成收益。
- 个股月度/年度收益率对齐券商 App 的持仓盈亏率口径：仍持仓标的用 `总盈亏 / 调整后持仓成本`，也就是 `总盈亏 / abs(期末市值 - 总盈亏)`；已清仓标的才回退到 `盈亏 / 投入本金`。
- 当前仍持仓标的的汇总收益率会优先对齐本轮持仓口径，清仓归零前的历史交易不会稀释新一轮仓位；同日 T 操作会先回冲本轮成本。
- 期权缺少本金/保证金时会按规则兜底：卖出认沽使用 cash-secured 资金口径，约等于 `行权价 × 乘数 × 张数 - 权利金 + 手续费`；买入期权按权利金加手续费；卖出认购只在表里有保证金或策略资金时采用，不自动虚构无限风险的本金。
- 盈亏金额对比会按账户资金规模折算 baseline，不再用指数自己的点位金额。
- 支持折线图和 K 线图。K 线图保留周 K、月 K、年 K，支持鼠标滚轮按时间轴缩放、拖动平移。
- K 线 tooltip 显示当前蜡烛的开盘、最高、最低、收盘、区间变化和振幅。
- 可选辅助分析：极值分析、最大增长、最大回撤；回撤区间用绿色，增长区间用黄色。
- 可以点击眼睛隐藏金额，只保留收益率。

## 使用方式

1. 打开 `Trade Tracker.xlsx`，可以先查看内置合成示例；正式使用前清空 `交易记录` 工作表里的示例行，再录入自己的交易。
2. 双击 `Update Preview.command` 启动本地预览服务。
3. 浏览器会打开本地看板；也可以手动打开 `Trade Tracker.html`。
4. 保持 `Update Preview.command` 弹出的终端窗口打开，网页里的“刷新看板”按钮才能调用本地刷新服务。

交易记录可以新增可选列 `标签`。多个标签用 `，`、`;`、`|`、空格或换行分隔，例如 `建仓，AI，事件驱动`。刷新后，交易时间线、盈亏日历和阶段账单会自动带出标签筛选。

工作簿和入口文件暂时保留 `Trade Tracker` 文件名，是为了兼容原始模板和现有启动脚本；项目名称统一为「韭菜账本 Leek Ledger」。

## 环境与启动

最推荐在 AI 编程工具里打开本文件夹，让 Codex、Claude 等工具帮你安装依赖、导入交易和刷新看板；这也是本项目最适合的使用方式，尤其适合多币种、多账户、交易记录来源比较分散的人。

最简单：

```text
双击 Update Preview.command
```

新的启动脚本会做这些自检：

- 检查 `Trade Tracker.xlsx`、生成器和预览服务是否存在。
- 自动创建 `.venv` 虚拟环境。
- 自动安装或补齐 `openpyxl`、`pandas`。
- 优先寻找能读取仓库内置看板核心的 Python；当前发布包建议使用 Python 3.14。
- 启动 `http://127.0.0.1:8765/preview/index.html`。
- 失败时给出中文错误和日志目录。

如果双击仍然失败，可以在终端里进入本目录运行：

```bash
./Update\ Preview.command
```

只想手动生成一次静态网页，可以运行：

```bash
.venv/bin/python .trade-tracker/tools/export_trade_tracker_html.py
```

想注册为 macOS 后台常驻服务，可以运行：

```bash
python3 .trade-tracker/tools/install_preview_service.py
```

后台服务成功后，网页里的刷新按钮不需要依赖终端窗口。

## 行情和缓存

刷新看板时会优先用公开在线行情源批量更新现股和期权价格：

- 现股优先走腾讯批量行情，东方财富和 Yahoo 补充缺失标的。
- 当日盈亏按市场本地自然日切换：A 股、港股按北京时间/香港时间当天，美股按纽约时间 0 点进入新一天；只有展示交易日前没有隔夜底仓、且展示交易日当天才建仓的标的，才按建仓价到现价计算；已有隔夜底仓的标的即使当天做 T 或买回，也继承行情源的昨收涨跌；美股现股优先读取 Yahoo 盘前/盘后价，拿不到扩展时段价时再回退到常规价或腾讯行情。
- 港股期权会尝试走 HKEX 延迟 Bid/Ask 中点。
- 指数历史行情会缓存到 `.trade-tracker/tools/cache/benchmark_history.json`，刷新时按指数增量补齐；旧的区间缓存会自动迁移并压缩成按指数保存的长期缓存。
- 个股真实历史行情会缓存到 `.trade-tracker/tools/cache/security_history.json`。

不需要安装或打开本地行情客户端。取不到行情时会尽量显示已有缓存或 `-`，不会阻塞整个页面生成。

看板会按币种分别展示总盈亏、收益率、持仓市值和当日盈亏，也会提供人民币折算汇总，适合把多个券商账户、多个市场放在同一套口径下复盘。

## 用 AI 导入券商交易

你可以把券商导出的 `.csv` / `.xlsx` 成交记录提供给 Codex、Claude 等 AI 工具，让它按下面流程整理：

1. 读取成交记录和费用字段。
2. 忽略银证转账、担保品划转、资金流水等非交易事件。
3. 识别股票、ETF、期权、分红和公司行动。
4. 补齐日期、代码、名称、数量、价格、费用、币种等字段。
5. 写入 `Trade Tracker.xlsx` 后刷新看板。

首次导入完成后，后续追加交易不一定要反复导出文件；把券商 App 或交易软件里的完整订单截图发给 AI 工具即可。截图里尽量包含方向、代码、名称、数量、价格、成交或下单时间、费用和币种，AI 工具就可以按工作簿字段继续追加记录。

如果使用真实交易数据，建议放在 private 仓库或本地私有目录里，不要提交到公开仓库。

## 个性化扩展

当前看板里的栏目、汇总口径和排版主要是作者个人偏好，不是固定答案。你完全可以继续让 Codex、Claude 等 AI 工具按自己的交易习惯改造，例如加入期权裸卖资金压力测试、cash-secured put 保证金/现金占用分析、不同汇总币种口径、账户分组、收益率计算方式、持仓排序和表格字段调整等。

## 目录说明

- `Trade Tracker.xlsx`: 带合成示例的交易记录模板，用来展示完整看板外观。
- `Trade Tracker.html`: 看板入口页。
- `Update Preview.command`: 推荐的一键启动入口，会自动检查虚拟环境和依赖。
- `.trade-tracker/tools/`: 看板生成和刷新脚本。
- `.trade-tracker/tests/`: 轻量单元测试，覆盖成本回冲等核心数据点。
- `.trade-tracker/preview/`: 由合成示例模板生成的静态预览。
- `skills/leek-ledger/SKILL.md`: 给 AI 助手使用的项目维护流程。

## 隐私说明

公开版本不会提交 `.trade-tracker/history/`、`.trade-tracker/logs/`、`.trade-tracker/tools/cache/` 或真实的 `security_name_cache.json`。如果你基于这个模板记录自己的交易，建议把包含真实数据的仓库设为 private。
